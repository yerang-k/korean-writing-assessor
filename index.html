AI 프록시 연결 (권장 · 선택)

프록시를 연결하면 API Key가 학생에게 전혀 전달되지 않습니다. 지금 방식(키를 공유 링크에 담아 전달)은 키가 노출될 수 있고 사용량이 소진될 위험이 있는데, Cloudflare Worker에 키를 숨겨두면 그 위험이 사라집니다.

🛡️ Cloudflare Worker 만드는 방법 보기 (클릭)
  1. dash.cloudflare.com 로그인 → Workers & Pages → Create → Workers → 이름 입력 후 Deploy
  2. Edit code 를 눌러 편집기를 열고, 기존 코드를 모두 지운 뒤 아래 코드를 붙여넣고 Deploy
  3. Worker의 Settings → Variables and Secrets 에서 다음 두 개를 Secret 으로 추가:
    GEMINI_API_KEY = 발급받은 Gemini 키  /  ACCESS_TOKEN = 원하는 문자열(예: barungeul-2026-1)
  4. 코드 맨 위 ALLOWED_ORIGIN 이 이 앱 주소와 같은지 확인 (다르면 수정 후 다시 Deploy)
  5. Worker 주소와 토큰을 위 칸에 입력하고 저장 → 끝!
⚠️ 프록시를 쓰기 시작하면 Google Cloud에서 그 키의 HTTP 리퍼러 제한은 반드시 '없음'으로 바꿔야 합니다. (서버에서 호출하므로 리퍼러가 없어 차단됩니다.) API 제한(Generative Language API만)은 그대로 유지하세요. 이미 학생에게 나간 키가 있다면 재발급 후 Worker에 넣는 것을 권장합니다.

  3. 기존 코드를 모두 지우고 아래의 Apps Script 코드를 복사해서 붙여넣습니다.

🔒 보안 강화 — AI 프록시 (학생 배포 시 권장)

공유 링크에 API Key를 포함하면, 링크를 받은 사람이 키를 꺼내 쓸 수 있고 사용량이 소진될 위험이 있습니다. AI 프록시를 쓰면 키가 Cloudflare Worker(서버) 안에만 있고 학생에게는 전달되지 않습니다.

💡 프록시를 쓰면 학생 링크에는 프록시 주소와 접근 토큰만 담깁니다. 토큰이 새어나가도 토큰만 바꾸면 되므로 API Key를 재발급할 필요가 없습니다.

설치 순서

  1. API Key 발급 — Google AI Studio(aistudio.google.com/apikey)에서 API 키 만들기. 이때 이미 쓰고 있던 기존 프로젝트를 선택하세요. 키를 만들 때 애플리케이션 제한사항은 '없음' 으로 둡니다.
  2. Worker 만들기 — dash.cloudflare.com → Workers & Pages → Create → Workers → 이름 입력 후 Deploy. 생성된 주소(…workers.dev)를 메모합니다.
  3. 코드 붙여넣기 — 이 앱 [설정 & API] → AI 프록시 연결 → 'Worker 만드는 방법 보기' 에서 코드 복사 → Worker의 Edit code 에서 기존 코드를 모두 지우고 붙여넣기 → Deploy.
  4. 비밀값 2개 등록 — Worker의 Settings → Variables and Secrets 에서 Secret 타입으로 추가:
    GEMINI_API_KEY = 발급받은 키  /  ACCESS_TOKEN = 직접 정한 암호 문자열
  5. 앱에 연결 — [설정 & API] → AI 프록시 연결에 Worker 주소와 토큰을 입력하고 저장. 왼쪽 아래가 'AI 프록시 연결됨' 으로 바뀌면 성공입니다.
  6. 테스트 & 마무리 — [교사 모드]에서 주제 추천을 눌러 작동을 확인한 뒤, 공유 링크를 새로 만들어 배포하고 예전 키는 삭제하세요.

꼭 주의할 3가지

① 새로 만든 프로젝트는 무료 할당량이 0일 수 있습니다. limit: 0 오류가 나면 기존에 쓰던 프로젝트에서 키를 다시 발급하세요.

② 키에 HTTP 리퍼러(웹사이트) 제한을 걸면 프록시가 차단됩니다. 프록시를 쓸 키는 애플리케이션 제한사항 = '없음' 이어야 합니다. (API 제한 = Gemini API만 은 걸어도 됩니다.)

③ Worker 코드 맨 위 ALLOWED_ORIGIN 이 이 앱 주소와 같아야 합니다. 다르면 "허용되지 않은 접근입니다"가 뜹니다.

오류가 날 때

"허용되지 않은 접근입니다"

Worker 코드의 ALLOWED_ORIGIN 이 앱 주소와 다릅니다. 수정 후 다시 Deploy.

"접근 토큰이 올바르지 않습니다"

앱에 입력한 토큰과 Worker의 ACCESS_TOKEN 값이 다릅니다. 공백·대소문자를 확인하세요.

"no longer available to new users"

그 모델을 쓸 수 없다는 뜻입니다. Worker가 사용 가능한 모델을 자동으로 찾으니, 최신 Worker 코드를 다시 붙여넣고 Deploy 해보세요.

"Quota exceeded … limit: 0"

그 프로젝트에 무료 할당량이 없습니다. 기존(무료 등급이 살아있는) 프로젝트에서 키를 새로 만들어 Worker의 GEMINI_API_KEY 를 교체하세요.

토큰이 유출된 것 같아요

Worker의 ACCESS_TOKEN 값만 새로 바꾸고 공유 링크를 다시 생성하면 됩니다. API Key는 그대로 두셔도 됩니다.

자주 묻는 질문 · 문제 해결

Q. 채점 버튼을 눌러도 오류가 납니다.

A. [설정 & API]에서 API Key를 다시 저장 및 검증해 보세요. 하루 무료 사용량을 모두 쓴 경우에도 실패할 수 있습니다.

Q. [평가 결과] 탭이 비어 있습니다.

A. 정상입니다. 이 화면은 지금 이 기기에서 제출한 글만 보여줍니다. 학생들의 결과는 [제출 현황] 탭에서 확인하세요.

Q. "제출 내역 로드 실패"가 뜹니다.

A. Apps Script 배포 설정을 확인하세요. ① 액세스: 모든 사용자 ② 주소가 /exec 로 끝남 ③ 코드를 고쳤다면 배포 관리 → 새 버전으로 다시 배포.

Q. 학생이 다른 기기로 접속하면 자기 기록을 볼 수 있나요?

A. 학생 화면의 과거 기록은 그 브라우저에만 남아 보이지 않습니다. 다만 제출한 내용은 선생님 시트에 안전하게 저장되어 있습니다. 학생에게는 제출 직후 다운로드로 사본을 받아두게 하세요.

Q. 같은 학생이 여러 번 제출하지 못하게 하려면?

A. [교사 모드]에서 '중복 제출 허용' 체크를 해제하세요. 이미 제출한 학생은 다시 제출할 수 없습니다. (구글 시트 연동 시 다른 기기로 바꿔도 차단됩니다.)

Q. 다른 선생님께 어떻게 알려주나요?

A. 앱 주소 뒤에 ?mode=teacher 를 붙인 교사용 주소를 알려주시고, 교사 비밀번호도 함께 전달해 주세요. 각 선생님이 자기 API Key와 시트를 등록해 쓰면 됩니다.
